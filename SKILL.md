# PDF 翻译为中文 Markdown

## 不可违背的默认规则

1. **默认使用视觉模式。** 直接使用当前 AI 模型的多模态能力读取 PDF 页面；不得默认调用 `pypdf`、`pdfplumber` 或其他文本提取器。
2. **页面渲染不等于文本提取。** 可以用 Poppler、PyMuPDF 等工具把 PDF 页面渲染为 PNG，再由当前 AI 模型识别和翻译。
3. **不得静默回退到文本提取。** 只有用户明确要求 Python、`pypdf`、`pdfplumber`、提取文本、纯文本或 `--input-mode text` 时，才使用文本模式。
4. **优先匹配期刊论文。** 默认逐页判断单栏、双栏、跨栏、图表和脚注布局，按视觉阅读顺序处理。
5. **翻译由当前 AI 模型完成。** 不绑定任何特定模型名称。
6. **不确定内容不得猜测。** 使用 `[原文模糊，待核对]`、`[公式符号待核对]` 等明确标记。

## 调用方式

```text
/pdf-trans2md <PDF路径> [选项]
```

常用选项：

- `--pages 1-10,20-30`：指定 1-based 页码范围。
- `--output <path>`：指定输出 Markdown。
- `--title <title>`：指定文档标题。
- `--mode zh|bilingual`：默认 `zh`。
- `--layout auto|single-column|double-column`：默认 `auto`。
- `--input-mode vision|text`：默认 `vision`；`text` 必须由用户明确指定。
- `--chapter <number-or-title>`：按目录、书签或视觉标题定位章节。
- `--no-toc`：不生成目录。
- `--no-structure`：不重建 Markdown 结构；**不代表文本提取模式**。
- `--keep-intermediate`：保留页面图像、裁剪图或文本模式中间文件。

兼容别名：`--text-extract` 等同于 `--input-mode text`，但同样必须来自用户明确意图。

## 模式选择

### 默认：视觉模式

以下请求均使用视觉模式：

- “翻译这个 PDF”
- “把论文转成中文 Markdown”
- “翻译第 3–8 页”
- “生成中英对照版本”
- 扫描 PDF、双栏论文或图表密集 PDF

即使 PDF 有可提取文本层，也不得因此自动改用文本模式。

### 显式：文本模式

只有出现明确意图时才可使用：

- “用 Python/pypdf/pdfplumber 提取文本后翻译”
- “不要识图，使用 PDF 文本层”
- “使用纯文本模式”
- 用户传入 `--input-mode text` 或 `--text-extract`

文本模式使用 `scripts/extract_pdf_text.py`。如果用户没有明确授权，而视觉工具暂时不可用，应说明阻碍或请求允许，不能静默调用文本提取器。

## 视觉模式工作流

### 1. 检查输入

- 验证文件存在、可读且是 PDF。
- 确定总页数和用户指定页码。
- 对加密 PDF，说明需要密码或解密版本。
- 对大文件按页批处理，不一次加载整篇。

### 2. 获取页面视觉输入

优先直接使用支持视觉读取的 PDF 输入；否则运行：

```text
python scripts/render_pdf_pages.py <pdf> --pages <range> --dpi 240
```

推荐设置：

- 普通页面：200–240 DPI。
- 字体很小的双栏论文：240–300 DPI。
- 先查看完整页面以判断版面，再按需裁剪栏、图或表；不要在不了解全页结构时直接裁栏。
- 普通双栏正文每批 2–4 页；复杂图表页每批 1 页。

### 3. 逐页建立版面图

每页先识别：

- 跨栏标题、作者、单位、摘要和关键词；
- 单栏、双栏或多栏正文区域；
- 图、表、公式、图注、表注和脚注；
- 页眉、页脚、页码、水印和重复期刊信息；
- 与上一页或下一页衔接的未完段落。

先确定阅读顺序，再识别和翻译文字。不要按照图像中的水平行坐标在左右栏之间交叉读取。

### 4. 翻译并持续写入

- 按章节和语义块翻译，而不是机械逐行翻译。
- 每完成一个页面批次即追加到 Markdown 草稿。
- 维护临时术语表、已完成页码、图表编号和跨页续接状态。
- 代码标识符、API、配置键、文件路径、DOI 和数学变量保持原样。

### 5. 视觉复核

每页检查：

- 是否遗漏任一栏；
- 左右栏是否顺序正确；
- 跨栏内容是否插入正确位置；
- 图注、表注是否误拼入正文；
- 页眉页脚是否被误收录；
- 跨页段落是否重复或断裂；
- 公式、单位、上下标和参考文献编号是否准确。

## 双栏期刊论文规则

### 标准阅读顺序

标准双栏页面通常按以下顺序处理：

1. 页面顶部跨栏内容；
2. 左栏从上到下；
3. 右栏从上到下；
4. 页面底部跨栏内容；
5. 脚注。

但必须以视觉版面为准。跨栏图表可能出现在两栏正文之间，首页也可能先有跨栏摘要再进入双栏正文。

### 栏裁剪策略

如果完整页面字体过小：

1. 先查看完整页并记录版面边界；
2. 分别生成左栏、右栏或局部图表裁剪；
3. 按已确定的阅读顺序识别裁剪图；
4. 回到完整页核对跨栏元素和段落衔接。

裁剪只用于提高识别清晰度，不能丢失完整页面提供的布局关系。

### 跨页和换栏断词

- 合并由换栏或分页造成的英文断词，如 `multi-` + `modal` → `multimodal`。
- 不合并真正的复合词。
- 上一页右栏末尾与下一页左栏顶部可能属于同一段，应合并为一个语义块。
- 不重复翻译跨页标题、图注或参考文献条目。

## 输出模式

### 中文模式 `--mode zh`

只输出中文，保留原文中的标题层级、列表、公式、图表编号和引用关系。

```markdown
## 3. 实验结果

所提出的方法在所有基准数据集上均取得了更好的性能。
```

### 双语模式 `--mode bilingual`

**强制格式：先原文，后中文译文；中文译文必须使用引用块。**

普通段落：

```markdown
The proposed method achieves better performance on all benchmark datasets.

> **译文：**
> 所提出的方法在所有基准数据集上均取得了更好的性能。
```

标题：

```markdown
## 3. Experimental Results

> **译文标题：3. 实验结果**
```

列表按完整列表成组对照：

```markdown
- Multimodal feature alignment
- Cross-domain generalization

> **译文：**
> - 多模态特征对齐
> - 跨领域泛化
```

禁止把英文原文放入引用块，也禁止把中文译文放在对应英文之前。

代码和公式通常只保留一份，不为满足双语形式而机械重复。

## 特殊内容处理

### 图和图注

- 保留 `Figure 1`、`Fig. 2` 等编号及其引用关系。
- 翻译图注，以及有语义价值的坐标轴、图例、面板标签和流程图节点。
- 图中文字无法可靠辨认时明确标记，不得臆造。
- 除非用户要求，不必把原图嵌入 Markdown；如需嵌入，使用稳定的相对路径和清晰文件名。

### 表格

- 根据视觉网格、对齐关系和表头层级重建 Markdown 或 HTML 表格。
- 复杂合并单元格无法可靠恢复时，可转为结构化列表并说明转换。
- 不得仅因表格复杂而未经授权切换到 Python 文本提取。

### 数学公式

- 简单公式转写为 LaTeX：行内 `$...$`，独立公式 `$$...$$`。
- 保留公式编号、变量、上下标、单位和符号；只翻译公式周围的解释文字。
- 无法确认的符号使用 `[公式符号待核对]`。

### 代码

- 代码块、命令、API、变量名和配置键保持原样。
- 默认不修改代码块内部，包括注释和字符串；需要翻译注释时在代码块外另作说明。

### 参考文献

- 默认保留作者、题名、期刊、年份、卷期、页码和 DOI 原样。
- 不默认翻译每条参考文献题名；用户明确要求时再翻译。
- 保持正文引用编号与参考文献列表一致。

### 页眉、页脚和水印

默认过滤重复的期刊名称、短标题、页码、数据库水印和下载声明。首次出现且具有文献意义的许可或版权信息可保留。

## 术语与翻译质量

- 首次出现的专业术语可采用“中文（English）”，后续保持统一译法。
- 根据学科语境翻译，不逐词硬译。
- 保留人名、软件名、模型名、基因名、化学式和正式标准名称。
- 不擅自补充原文没有的结论、因果关系或解释。
- 长文使用临时术语表；除非用户要求，不必把术语表附在最终文档中。

## 输出与中间文件

- 默认输出：`<原文件名>_zh.md` 或 `<原文件名>_bilingual.md`。
- 用户指定 `--output` 时优先使用指定路径。
- 页面图像和裁剪图放在任务临时目录；成功后清理，除非指定 `--keep-intermediate`。
- 可在草稿中使用 `<!-- PDF page: 12 -->` 追踪来源；最终是否保留由任务需要决定。
- 不在最终文档中留下工具日志、模型说明或内部检查清单。

## 失败与回退

- 页面无法渲染：报告缺失的渲染依赖或文件问题。
- 页面模糊：提高 DPI、局部裁剪或请求更清晰 PDF。
- PDF 加密：请求密码或解密版本。
- 复杂版面无法确认：标记不确定区域并请求人工核对。
- **视觉模式失败时，不得自动使用文本提取。** 只有用户明确同意后，才切换到 `--input-mode text`。

## 完成标准

交付前确认：

- 所有目标页均已处理且无重复；
- 双栏顺序正确，没有左右栏交叉；
- 双语模式始终先原文后译文，中文译文位于引用块；
- 图表、公式、图注、表注和脚注位置合理；
- 术语、编号、引用和 Markdown 层级一致；
- 不确定内容已明确标记；
- 未在缺乏明确授权时调用文本提取后端。
