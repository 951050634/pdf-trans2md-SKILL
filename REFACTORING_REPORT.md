# pdf-trans2md v3.0 重构报告

## 重构日期

2026-07-14

## 目标

将 Skill 从旧版“Python 默认提取文本，再交给特定模型翻译”的架构，重构为：

```text
PDF 页面 → 视觉输入 → 当前 AI 模型多模态理解 → 版面排序 → 翻译 → Markdown
```

## 主要变化

### 1. 模型中立

- 删除所有绑定特定模型品牌或任务名称的表述。
- 统一使用“当前 AI 模型”或“当前多模态模型”。
- 翻译流程不依赖模型品牌。

### 2. 默认路径反转

旧版：

```text
pypdf.extract_text() → 正则结构检测 → AI 翻译
```

新版：

```text
PDF 页面渲染 → AI 视觉识别 → 版面与阅读顺序分析 → AI 翻译
```

`pypdf` 仅保留为用户明确要求的文本提取后端。

### 3. 双栏论文成为一等场景

新增规则覆盖：

- 页面级单栏/双栏/跨栏判断；
- 左栏后右栏的自然阅读顺序；
- 跨栏标题、摘要、图表和公式；
- 图注、表注、脚注和页眉页脚；
- 跨页段落与换栏断词；
- 先查看完整页、再按需裁栏的策略。

### 4. 双语输出契约变化

新版强制：

```markdown
Original text.

> **译文：**
> 中文译文。
```

原文必须在前，中文译文必须放入 Markdown 引用块。旧版相反顺序不再允许。

### 5. 辅助脚本重新分工

- `scripts/render_pdf_pages.py`
  - 默认视觉流程的页面渲染器；
  - 不提取文本；
  - 支持 PyMuPDF/Poppler、页码范围、DPI 和左右栏裁剪。

- `scripts/extract_pdf_text.py`
  - 旧脚本降级后的显式文本模式工具；
  - 强制要求 `--input-mode text`；
  - 使用 `pypdf` 提取并结构化文本；
  - 当前 AI 模型仍负责翻译。

## 行为护栏

- PDF 有文本层不代表可以自动使用文本模式。
- 视觉模式失败时不得静默回退。
- 文本模式必须来自用户明确意图。
- 页面渲染是视觉输入准备，不属于文本提取。
- 不确定的文字、表格和公式必须标记，不得臆造。

## 兼容性

保留：

- `--pages`
- `--output`
- `--title`
- `--mode zh|bilingual`
- `--chapter`
- `--no-toc`
- `--no-structure`
- `--keep-intermediate`

新增：

- `--input-mode vision|text`
- `--text-extract` 兼容别名
- `--layout auto|single-column|double-column`

说明：这些是 Skill 层调用约定；渲染和文本辅助脚本只实现各自所需的子集。

## 验收重点

- 默认请求不得调用 `extract_text()`。
- 双栏阅读顺序不得左右交叉。
- 双语输出必须先原文后中文引用块。
- 扫描 PDF 可直接进入视觉流程。
- 图表、公式和图注应按视觉关系处理。
- 仓库中不得残留把当前架构描述为特定模型优先或 Python-first 的活动文档。

## 后续可选增强

- 自动生成页面布局清单；
- 根据完整页视觉分析生成精确裁剪坐标；
- 增加更多学科的术语一致性评测；
- 为跨栏复杂表格增加专门的 HTML 输出模板。
